# Project instructions

This project generates standardized survey report PDFs from Quarto.

## Main files
- `report.qmd` = main report source- contains the quarto yaml and then content written in HTML. It also contains R code that processes data and generates plots inside the report.
- `styles-base.css` = base styling fine-tuned to match the target report visuals exactly. Contains styles for the first few slides
- `styles-ai.css` = styles for the majority of slides, generated by AI, that need a lot of polishing.
- `Rkod` = data-processsing R logic
- `data/` = input survey data
- `pdf_output/` = rendered outputs
- `fonts/` = local fonts required for rendering
- 'GRAFICKY_NAVRH_Engagement' = the target report that will be recreated via Quarto.

## Goals
- Create a Shiny interface that would help me with styling the final quarto report.
- The aim of the shiny app is to allow me to change a graphical parameters quickly, and streamline the fine-tuning of the report's visual.


## Rules
- Ask before deleting files.
- Do not modify files outside this folder.
- Write all new outputs to a clearly named new subfolder.
- Treat visual fidelity as high priority.
- Prefer incremental changes and explain the render pipeline before major refactors.
-Do not modify navratnost_api.R, token.txt or report_editable_api.qmd files!
